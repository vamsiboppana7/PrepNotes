Angular lifecycle hooks give you fine-grained control over your components. Use them wisely to manage setup, updates, and cleanup in your application.

üìå Introduction

Angular v17 introduces a new "developer preview" feature called "control flow syntax". This feature allows you to use a new template syntax to write control flow statements, like if/else, for, and switch, instead of using the built-in structural directives (`*ngIf`, `*ngFor`, and `*ngSwitch`).

üìå Structural Directives Under the Hood

Structural directives are directives that change the structure of the DOM by adding, removing, or manipulating elements. They are easy to recognize in Angular because they begin with an asterisk `*`.

For example:


        <h1>Ninja Squad</h1>
        <ul *ngIf="condition">
        <li *ngFor="let user of users">{{ user.name }}</li>
        </ul>

Angular decomposes the component‚Äôs template into ‚Äúviews‚Äù. A view is a fragment of the template that has static HTML content. It can have dynamic attributes and texts, but the HTML elements are stable.

üìå Custom Structural Directives

You can create your own structural directives if you want to. For example, a `*customNgIf` directive:


        import { Directive, DoCheck, EmbeddedViewRef, Input, TemplateRef, ViewContainerRef } from '@angular/core';
        
        @Directive({
        selector: '[customNgIf]',
        standalone: true
        })
        export class CustomNgIfDirective implements DoCheck {
        @Input({ required: true, alias: 'customNgIf' }) condition!: boolean;
        conditionalView: EmbeddedViewRef | null = null;
        
        constructor(private vcr: ViewContainerRef, private tpl: TemplateRef) {}
        
        ngDoCheck() {
            if (this.condition && !this.conditionalView) {
            this.conditionalView = this.vcr.createEmbeddedView(this.tpl);
            } else if (!this.condition && this.conditionalView) {
            this.conditionalView.destroy();
            this.conditionalView = null;
            }
        }
        }
üìå Limitations of Structural Directives

Structural directives have some drawbacks:

  • Clunky syntax for `else` alternatives.
  • Type-checking issues, especially with `*ngIf` and `*ngSwitch`.
  • Dependence on `ngDoCheck`, which is tied to `zone.js`.
üìå Control Flow Syntax

The control flow syntax is a new syntax introduced in Angular v17 to write control flow statements in templates. The syntax is very similar to some other templating syntaxes you may have met in the past, and even to JavaScript itself.

For example:


        <h1>Ninja Squad</h1>
        @if (condition) {
        <ul>
            @for (user of users; track user.id) {
            <li>{{ user.name }}</li>
            }
        </ul>
        }
üìå If Statement

With the control flow syntax, you can write `if/else` statements more intuitively:


        @if (condition) {
        <div>condition is true</div>
        } @else if (otherCondition) {
        <div>otherCondition is true</div>
        } @else {
        <div>condition and otherCondition are false</div>
        }
üìå For Statement

For loops are written with the `@for` syntax:


        <ul>
        @for (user of users; track user.id) {
            <li>{{ user.name }}</li>
        }
        </ul>

Note that specifying a `track` property is mandatory for performance reasons.

üìå Empty Clause

Handle empty collections with the `@empty` clause:


        <ul>
        @for (user of users; track user.id) {
            <li>{{ user.name }}</li>
        } @empty {
            <li>No users</li>
        }
        </ul>
üìå Loop Variables

Access loop variables like `$index`, `$first`, `$last`, `$even`, `$odd`, and `$count` directly:


        <ul>
        @for (user of users; track user.id; let isOdd = $odd) {
            <li [class.grey]="isOdd">{{ $index }} - {{ user.name }}</li>
        }
        </ul>
üìå Performance Considerations

The control flow syntax uses new compiled instructions under the hood, making it more performant than traditional structural directives. The required `track` property in `@for` loops ensures better performance by avoiding unnecessary intermediate maps.

üìå Conclusion

Angular's new control flow syntax offers a more intuitive and performant way to handle control flow in templates. It simplifies the syntax, improves type-checking, and enhances performance, making it a valuable addition for Angular developers.
